# Kadaba Document Generator v2

Internal document generation tool for Kadaba Construction Ltd.
Built with **Bun + TypeScript + Elysia** on the backend, and a plain **HTML/CSS/JS** frontend.
PDFs are generated by rendering HTML templates (Handlebars) through Puppeteer (headless Chrome),
so you can edit the layout directly in `src/templates/` without touching any code.

## Setup

### 1. Install dependencies

```bash
cd backend
bun install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env`:

| Variable | Description |
|---|---|
| `PORT` | API port (default: 3000) |
| `ALLOWED_ORIGIN` | Frontend URL for CORS, e.g. `http://localhost:5500` |
| `ANTHROPIC_API_KEY` | Your Anthropic API key (for AI letter body) |
| `SMTP_HOST` | SMTP server, e.g. `smtp.gmail.com` |
| `SMTP_PORT` | `587` (TLS) or `465` (SSL) |
| `SMTP_USER` | SMTP login email |
| `SMTP_PASS` | Gmail App Password or SMTP password |
| `SMTP_FROM` | Sender name + email |
| `GENERATED_DIR` | Where PDFs are saved (default: `./generated`) |
| `CHROMIUM_PATH` | Path to Chrome/Chromium binary (auto-detected if blank) |

### 3. Start the backend

```bash
bun run dev      # development (auto-restart on file change)
bun run start    # production
```

You should see:
```
╔══════════════════════════════════════════════════╗
║   Kadaba Document Generator API                  ║
║   Version: 2.0.0  |  Runtime: Bun + Elysia      ║
║   Listening on http://localhost:3000              ║
╚══════════════════════════════════════════════════╝
```

### 4. Open the frontend

Open `frontend/index.html` in any browser — no build step needed.

> **Important:** If you deployed the backend on a different URL, update the `API` constant
> at the top of the `<script>` in `frontend/index.html`:
> ```js
> const API = 'https://api.kadabaconstruction.com/api';
> ```

## API Reference

### `GET /api/health`
Returns service status.

### `GET /api/documents/counters`
Returns next receipt number (used by the UI to show a preview).
```json
{ "nextReceipt": "0016", "nextAllocation": 1 }
```

### `POST /api/documents/allocation`
Generates an Allocation Letter PDF.

**Body:**
```json
{
  "buyer": {
    "name": "Haruna Attai Usman",
    "phone": "+2348037960842",
    "email": "usmanharuna7500@gmail.com",
    "address": "House 10, 14th Estate Lifecamp, Abuja",
    "photoBase64": "data:image/jpeg;base64,..."
  },
  "property": {
    "description": "FIVE (5) BEDROOM FULL DETACHED DUPLEX",
    "blockNo": "001",
    "estate": "Khira's Villa",
    "location": "Gudu, Abuja",
    "plotNo": "Plot No. 95",
    "cadastralZone": "Cadastral Zone B14",
    "district": "Duste"
  },
  "financials": {
    "offerPrice": 80000000,
    "discount": 20000000,
    "vatPct": 5,
    "durationText": "Ten (10) Months",
    "deposit": 30000000
  }
}
```

**Response:**
```json
{
  "success": true,
  "ref": "KAD/GUDU/2026/001",
  "filename": "AllocationLetter_KAD_GUDU_2026_001_1717689600000.pdf",
  "downloadUrl": "/api/documents/download/AllocationLetter_KAD_GUDU_2026_001_1717689600000.pdf"
}
```

### `POST /api/documents/receipt`
Generates a Payment Receipt PDF.

**Body:**
```json
{
  "payer": {
    "name": "Onwuka Anthony Amaobi",
    "phone": "08032003320",
    "email": "anthony@email.com",
    "address": "House No:3 Ngene Ajiri Road, Amuvi, Abia State"
  },
  "purpose": "(3) THREE BEDROOM BUNGALOW BLOCK NO: S3651 ED-C1 KUJE, FCT ABUJA",
  "amount": 1500000,
  "method": "Bank Transfer",
  "date": "2025-09-29",
  "bankRef": "Access Bank — TXN987654",
  "balance": "₦ _____________",
  "notes": ""
}
```

**Response:**
```json
{
  "success": true,
  "receiptNo": "0016",
  "filename": "Receipt_0016_Onwuka_Anthony_Amaobi_1717689600000.pdf",
  "downloadUrl": "/api/documents/download/Receipt_0016_Onwuka_Anthony_Amaobi_1717689600000.pdf"
}
```

### `GET /api/documents/download/:filename`
Streams a generated PDF for download.

### `POST /api/email/allocation`
Emails an allocation letter to the buyer.
```json
{
  "recipientEmail": "buyer@email.com",
  "recipientName": "Haruna Attai Usman",
  "filename": "AllocationLetter_KAD_GUDU_2026_001_...",
  "ref": "KAD/GUDU/2026/001"
}
```

### `POST /api/email/receipt`
Emails a receipt to the payer.
```json
{
  "recipientEmail": "payer@email.com",
  "recipientName": "Onwuka Anthony Amaobi",
  "filename": "Receipt_0016_...",
  "receiptNo": "0016"
}
```

## Customising the Document Layout

The PDF layout lives entirely in two HTML files — no code changes needed for layout tweaks:

| File | What it controls |
|---|---|
| `src/templates/allocation.html` | Full Allocation Letter layout, fonts, spacing, table |
| `src/templates/receipt.html` | Full Receipt layout, amount box, checkboxes |

Templates use **Handlebars** syntax: `{{variable}}`, `{{#if condition}}…{{/if}}`.

### Adding the company logo

In both templates, find this block:
```handlebars
{{#if logoBase64}}
  <img src="{{logoBase64}}" ...>
{{else}}
  <!-- text fallback -->
{{/if}}
```

In `pdfService.ts`, load your logo and pass it as base64:
```ts
import { readFileSync } from 'fs';
const logoBase64 = 'data:image/png;base64,' + readFileSync('./assets/logo.png').toString('base64');
// then pass logoBase64 into the templateData object
```

## Receipt & Ref Number Logic

- **Receipt numbers** start at `0016` and increment atomically per generation.
- **Allocation refs** are built as `KAD/{LOCATION}/{YEAR}/{BLOCK_NO}`.
- Both are stored in `counter_store.json` (auto-created on first run).
- Numbers are **non-duplicating**: the counter increments before the PDF is written.
- For multi-server production, replace `counterStore.ts` with a PostgreSQL sequence or Redis atomic increment.

1. Enable 2-Factor Authentication on the Gmail account.

3. Create an App Password for "Mail".
4. Use the 16-character password as `SMTP_PASS` in `.env`.

## Deployment

### Option A — VPS with PM2
```bash
# Install PM2
bun add -g pm2

# Start
pm2 start --name kadaba-docs --interpreter bun src/server.ts

# Auto-restart on reboot
pm2 startup && pm2 save
```

Put Nginx in front as a reverse proxy on port 443 with SSL.

### Option B — Railway / Render / Fly.io
1. Push the `backend/` folder as a Bun service.
2. Set all environment variables in the platform dashboard.
3. Set the start command to `bun src/server.ts`.
4. Deploy — update `API` in `frontend/index.html` to your public URL.

## Future: Migrating to Vue

When you're ready to move the frontend to Vue:

1. `bun create vue@latest kadaba-frontend`
2. The backend API stays exactly the same — just update the `API` base URL in your Vue env file.
3. Move the form logic from `index.html` into Vue components:
   - `AllocationForm.vue`
   - `ReceiptForm.vue`
   - `DocumentModal.vue`
4. Use `axios` or `fetch` to call the same endpoints.

## Troubleshooting

**"Chromium not found"**
→ Set `CHROMIUM_PATH` in `.env` to the absolute path of your Chrome/Chromium binary.
→ Ubuntu: `which chromium-browser`; macOS: `/Applications/Google Chrome.app/Contents/MacOS/Google Chrome`

**PDF generation is slow**
→ Puppeteer launches a new browser per request. For production, use a persistent browser instance (Puppeteer browser pool) to cut generation time from ~3s to ~0.5s.

**CORS errors**
→ Set `ALLOWED_ORIGIN` in `.env` to exactly match the URL you're opening `index.html` from (including port).
→ During development with VS Code Live Server, this is typically `http://127.0.0.1:5500`.
